# 🚀 Chiru AI – Multi-Agent Intelligence Dashboard

> **One dashboard. 20 AI agents. Infinite possibilities.**

---

## 🧠 Overview

**Chiru AI** is a powerful **multi-agent AI system** that combines 20 specialized AI agents into a single intelligent dashboard.

Instead of relying on one general-purpose AI, Chiru AI works like a **team of experts**, where each agent is trained for a specific role such as:

* 💻 Code Review
* 📊 Financial Analysis
* 📢 Marketing Strategy
* 🧑‍💼 HR Assistance
* 📚 Research & Reporting

All agents collaborate to deliver **accurate, structured, and actionable outputs**.

---

## ⚙️ Key Features

### 🔹 Multi-Agent System

* 20 specialized AI agents
* Each agent performs a dedicated role
* No overlap, high accuracy

### 🔹 Smart Agent Routing

* Automatically detects user intent
* Routes tasks to the best agent

### 🔹 Structured Outputs

Every response includes:

* Key Insights
* Actionable Steps
* Clean formatting

### 🔹 Multi-Agent Collaboration

Complex workflows are handled using multiple agents:

```
Research → Data → Summary → Report
```

### 🔹 Unified Dashboard

* Single interface for all agents
* Easy switching between tasks
* Clean UI experience

## 🏗️ System Architecture

```
Frontend (React / Flutter / SwiftUI)
        ↓
Agent Router (Node.js / FastAPI)
        ↓
Prompt Engine (Agent Selection)
        ↓
LLM API (OpenAI / Gemini)
        ↓
Response Formatter
        ↓
Dashboard UI
```

---

## 🧠 How It Works

1. User enters a query
2. System identifies intent
3. Best agent is selected
4. Agent processes request
5. Structured response is returned

## 🚀 Getting Started

### 🔧 Prerequisites

* Node.js / Python
* OpenAI API Key or Gemini API Key
* Git

---

### 📦 Installation

```bash
git clone https://github.com/your-username/chiru-ai.git
cd chiru-ai
npm install
```

---

### ▶️ Run the Project

```bash
npm start
```

or (backend)

```bash
uvicorn main:app --reload
```

---

## 🔑 Environment Variables

Create a `.env` file:

```
OPENAI_API_KEY=your_api_key_here
GEMINI_API_KEY=your_api_key_here
```
